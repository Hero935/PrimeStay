# 🏠 高端租屋代管平台｜角色權限與加入流程規格 (roles.md)

## 1. 角色定義概述

本系統採用雙層權限架構：
- **系統角色 (System Role)**: 決定用戶在全系統層級的基礎權限。
- **組織角色 (Member Role)**: 決定用戶在特定組織 (Organization) 內的管理權限。

### 1.1 系統角色 (System Role)
對應資料庫 `User` 資料表的 `systemRole` 欄位。

| 角色名稱 | 列舉值 (Enum) | 跨組織存取 | 核心職責 |
| :--- | :--- | :--- | :--- |
| **平台管理員** | `ADMIN` | 是 | 系統初始化、環境變數管理、監控全平台運行、手動處理爭議案件。 |
| **房東** | `LANDLORD` | 否 | 組織創立者，擁有房源資產，負責財務審核與人員授權。 |
| **代管人員** | `MANAGER` | 否 | 執行日常管理業務，如房源維護、租約簽署、帳單催收。 |
| **房客** | `TENANT` | 否 | 最終使用者，負責度數回報、繳費、報修。 |

### 1.2 組織成員角色 (Member Role)
對應資料庫 `UserOrganization` 資料表的 `memberRole` 欄位。

- **OWNER**: 組織擁有者（通常為 Landlord），擁有該組織的所有管理權限。
- **MANAGER**: 組織管理員（通常為 Manager），擁有受限的管理權限，無法刪除組織或變更擁有者。

## 3. 註冊與加入流程

### 3.1 房東 (Landlord) 流程
1. **觸發**: 由 `ADMIN` 透過後台發送邀請郵件。
2. **註冊**: 房東點擊邀請連結，填寫基本資料。
3. **初始化**: 系統自動為該房東建立一個專屬的 `Organization`，並將該房東設為 `OWNER`。

### 3.2 代管人員 (Manager) 流程
1. **觸發**: 由 `LANDLORD` 在組織管理頁面生成邀請邀請連結。
2. **註冊**: 代管人員登入或註冊帳號。
3. **繫結**: 系統自動將該用戶加入該 `Organization`，並賦予 `memberRole: MANAGER` 與 `systemRole: MANAGER`。

### 3.3 房客 (Tenant) 流程
1. **觸發**: 房東或代管人員針對特定「房源 (Property)」生成邀請碼。
2. **註冊**: 房客進入註冊頁面，輸入邀請碼。
3. **綁定**: 
   - 系統驗證邀請碼有效性。
   - 建立房客帳號 (`systemRole: TENANT`)。
   - **自動建立租約 (Contract)** 關聯，將房客與房源、組織連結。
4. **限制**: **嚴格禁止**無邀請碼的自主註冊，以確保高端代管的封閉性與資料正確性。

## 4. 註解與技術限制 (Developer Notes)

- **唯一性限制**: 一個 `User` 帳號在目前的實作中，只能對應一個主要 `systemRole`。
- **邀請碼安全性**: 邀請碼內嵌 `PropertyID` 與 `OrganizationID` 的加密資訊，並設有有效期 (`INVITATION_EXP_DAYS`)。
- **自動化邏輯**: 房客註冊完成的瞬間，必須確保 `Contract` 狀態從 `PENDING` 轉為 `ACTIVE` 或直接建立有效合約，不可出現孤立房客。
