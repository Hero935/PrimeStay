# 📋 PrimeStay SaaS 訂閱模型與 UI/UX 強化任務報告

## 1. 任務執行摘要
本階段開發精確遵循 SaaS 營運策略，將「組織 (Organization)」確立為計費與授權的核心，並成功實作了多層級角色權限 (RBAC) 與房源配額限制。同時，針對未登入訪客優化了房源探索體驗，並強化了後台 API 的安全性。

## 2. 完成子任務清單

### 🎨 前端 UI/UX 強化
- **方案儀表板**: 於 Sidebar 側邊欄實作 `PlanUsageProgress` 元件，即時監控組織房源配額（Free: 2, Starter: 10, Pro: 50）。
- **配額阻斷**: 當房源達到上限時，房東後台「建立房源」按鈕自動禁用並顯示提醒。
- **Pro 視覺識別**: 針對 Pro 方案組織之房源，於列表與詳情頁增加 Amber 色系之 Crown 標誌。
- **Admin 邀請升級**: 更新邀請表單，允許系統管理員在發送邀請時預設 Organization 之訂閱方案。

### 🌍 公開探索頁面
- **探索中心 (`/properties/explore`)**: 提供未登入用戶瀏覽所有標記為 AVAILABLE 的公開房源。
- **混合視角詳情頁 (`/properties/[id]`)**: 
    - **訪客**: 僅能預覽照片、地址摘要與月租金，敏感資訊（如房號、專屬聯繫方式）將被隱藏，並導向登入。
    - **成員**: 登入且具權限後，可查看完整水電配置、管理費詳情與直接聯繫按鈕。

### 🛡️ 權限與安全性加固
- **API Guard**: 封裝 `withAuth` 通用裝飾器，整合「登入校驗」、「角色權限 (Role)」與「帳號狀態 (Status)」檢查。
- **停權機制**: 任何 `status === 'SUSPENDED'` 的用戶將在進入 API 時即刻被 403 攔截。
- **數據隔離**: 確保 API 請求僅能獲取所屬組織內之資料。

### 📈 方案升級模擬
- **方案價格頁 (`/pricing`)**: 實作三層級方案對比與購買按鈕。
- **在線升級 API**: 模擬購買流程，成功後自動更新 Organization 方案等級並展延訂閱效期。

## 3. 技術債與建議
1. **地圖整合**: 公開頁面目前僅有地址文字，建議整合 Google Maps API 顯示區域範圍。
2. **付款 API**: 目前為模擬邏輯，進入正式環境需對接 Stripe 或綠界支付。
3. **地址脫敏**: 訪客視角可進一步模糊化地址細節，僅顯示到「路/段」以保障隱私。

## 4. Checkpoint
- **Checkpoint**: 完成 SaaS 訂閱基礎建設與公開探索功能。
- **說明**: 已實現從方案限制到訪客瀏覽的完整閉閉環。
